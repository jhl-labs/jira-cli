jira-cli

셀프호스팅 Jira Server/Data Center 환경에서 AI 에이전트와 자동화 스크립트가 이슈를 다루기 위한 가볍고 의존성 없는 CLI입니다. 단일 바이너리, 크로스플랫폼. A small, dependency-free CLI that lets AI agents and automation work with issues on a self-hosted Jira Server/Data Center instance. Single binary, cross-platform.

Go · 0 dependencies PAT / Basic auth JQL search Workflow transitions JHL License · non-commercial
$ curl -fsSL https://jhl-labs.github.io/jira-cli/install.sh | sh
$ jira-cli search --project PROJ --status "In Progress" --output text

왜 필요한가Why

공식 Atlassian Remote MCP는 Cloud 중심입니다. The official Atlassian Remote MCP is Cloud-first.

Server/Data Center는 인증·네트워크·권한 모델이 다릅니다. jira-cli는 그 간극을 메우는 전용 CLI 계층으로, 인증(PAT/Basic)·재시도·JQL 검색·워크플로 전이를 일관되게 처리해 에이전트가 안전하게 내부 Jira를 다루게 합니다. confluence-cli의 형제 프로젝트입니다. Server/Data Center differs in auth, networking, and permissions. jira-cli is a dedicated CLI layer that bridges the gap — handling auth (PAT/Basic), retries, JQL search, and workflow transitions. A sibling of confluence-cli.

기능Features

에이전트 친화적으로 설계됨Designed for agents

JQL 검색JQL search

간편 플래그(project/status/text) 또는 원시 JQL로 이슈 검색. Find issues with simple flags or raw JQL.

이슈 수명주기Issue lifecycle

create / get / update / comment / delete — 전 과정을 CLI로. create / get / update / comment / delete from the CLI.

워크플로 전이Workflow transitions

가능한 전이 조회 후 이름/ID로 적용, 전이 중 댓글·담당자 지정. List then apply transitions by name/id, with comment & assignee.

담당자 · 라벨Assign & label

담당자 지정/해제, 라벨 추가/삭제로 이슈 정리. Assign/unassign and add/remove labels.

에이전트 스킬 생성Agent skill generator

generate-skill로 claude/codex/gemini/opencode용 스킬 문서 생성. generate-skill emits skill docs for claude/codex/gemini/opencode.

견고한 호출Resilient calls

429/5xx에 지수 백오프 재시도, JSON 우선 출력. Exponential-backoff retries on 429/5xx, JSON-first output.

명령어Commands

환경변수로 인증을 분리Auth via environment

# 인증 환경변수Authentication
export JIRA_BASE_URL="https://jira.example.com"
export JIRA_TOKEN="<personal-access-token>"
export JIRA_PROJECT="PROJ"   # 기본 프로젝트(선택)default project (optional)
searchJQL로 이슈 검색find issues with JQL
jira-cli search --project PROJ --status "In Progress" --max 20
jira-cli search --jql 'assignee = currentUser() AND resolution = Unresolved'
get이슈 조회read an issue
jira-cli get --key PROJ-123 --output text
jira-cli get --key PROJ-123 --description
create이슈 생성create an issue
jira-cli create --project PROJ --summary "Fix login bug" --type Bug --priority High
transition워크플로 전이move through the workflow
jira-cli transition --key PROJ-123              # list transitions
jira-cli transition --key PROJ-123 --to "Done" --comment "Shipped"
assign담당자 지정/해제set/clear assignee
jira-cli assign --key PROJ-123 --assignee alice
jira-cli assign --key PROJ-123 --unassign
more댓글 · 라벨 · 삭제 · 프로젝트comment · labels · delete · projects
jira-cli comment --key PROJ-123 --body "Looking into this."
jira-cli labels --key PROJ-123 --add "backend,urgent"
jira-cli delete --key PROJ-123 --yes
jira-cli projects

주의Heads up

설명/댓글은 위키 마크업 텍스트Descriptions are wiki-markup text

Server/DC에서 이슈 설명과 댓글 본문은 위키 마크업 텍스트 문자열입니다 — 마크다운도 Cloud의 ADF JSON도 아닙니다. 일반 문자열로 보내면 됩니다. On Server/DC, issue descriptions and comment bodies are wiki-markup text strings — not Markdown and not Cloud's ADF JSON. Send plain strings.
